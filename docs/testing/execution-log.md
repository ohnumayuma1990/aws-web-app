# テスト実行記録

## テストケース定義

### TC-FE-01: フロントエンド基本レンダリング
- **概要**: アプリケーションが正常に起動し、主要な要素が表示されることを確認する。
- **実施方法**: `cd frontend && CI=true npm test`
- **期待結果**: "サーバーレス対戦ゲーム" というテキストが画面に表示され、テストがパスする。

### TC-BE-01: バックエンド基本ロジック
- **概要**: WebSocket接続、切断、およびルーム作成・参加・メッセージ送信の基本ロジックを検証する。
- **実施方法**: `cd backend && npm test`
- **期待結果**: Jestによる全てのユニットテストがパスすること。

### TC-IF-01: インフラ構成検証
- **概要**: CDKにより定義されたAWSリソース（DynamoDB, Lambda, API Gateway）が正しく構成されているか検証する。
- **実施方法**: `cd infra && npm test`
- **期待結果**: CDK Assertionによる全てのテストがパスすること。

### TC-INT-01: モックバックエンド統合テスト (ルーム作成・参加・通信)
- **概要**: モックサーバーを使用して、ルーム作成、ルーム参加、メッセージ送受信の一連フローを確認する。
- **実施方法**:
    1. モックバックエンドを起動 (`node mock-backend/server.js`)
    2. フロントエンドを起動 (`REACT_APP_WSS_URL=ws://localhost:8080 npm start`)
    3. 検証スクリプトを実行 (`node mock-backend/verify-frontend.js`)
- **期待結果**: スクリプトが "Integration test passed successfully." と出力し、正常終了する。

### TC-LD-01: 負荷テスト (同時接続・遅延計測)
- **概要**: 大量の同時接続がある環境下でのサーバーの挙動とメッセージ配信遅延を確認する。
- **目標**: 50クライアントの同時接続、メッセージ配信遅延 200ms 以内。
- **実施方法**:
    1. モックバックエンドを起動 (`node mock-backend/server.js`)
    2. 負荷テストスクリプトを実行 (`node mock-backend/load-test.js`)
- **期待結果**: 50接続が正常に確立され、平均遅延が目標値以内に収まること。
