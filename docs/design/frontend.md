# フロントエンド設計 (React)

本ドキュメントでは、フロントエンドのコンポーネント構成と状態管理について記述します。

## 1. 全体構成

### 1.1 技術スタック
- **React 18**: UIフレームワーク。
- **TypeScript**: 型定義の導入。
- **WebSocket API**: ブラウザ標準の通信インターフェース。
- **CSS**: スタイリング。

### 1.2 エントリーポイント
`frontend/src/index.tsx`: アプリケーションのルート。
`frontend/src/App.tsx`: 全てのビジネスロジックとUIコンポーネントが集約されたメインコンポーネント。

---

## 2. コンポーネント設計 (App.tsx)

### 2.1 ステート管理
- `connected`: WebSocketの接続状態。
- `roomId`: 現在参加中のルームID（空の場合はロビー画面を表示）。
- `users`: 現在のルーム内のプレイヤー接続IDリスト。
- `gameState`: 山札数、場のカード、ターン、ゲーム状態（waiting, playing, ended）などを保持。
- `hand`: 引いた自分の手札。
- `scores`: 各プレイヤーのスコア。
- `messages`: チャット履歴。
- `timeLeft`: ターンの残り時間（30秒からのカウントダウン）。

### 2.2 WebSocket 接続管理
- **useEffect (Mount/Unmount)**:
  - マウント時に `new WebSocket(WSS_URL)` で接続し、イベントリスナー（`onopen`, `onmessage`, `onclose`）を設定します。
  - アンマウント時に `ws.current.close()` で切断します。
- **useRef (ws.current)**:
  - 再レンダリング間で WebSocket インスタンスを永続化します。

---

## 3. 主要なユーザーインターフェース (UI)

### 3.1 ロビー画面 (roomId が空の場合)
- **ルーム作成**: 公開/プライベートを選択してルームを作成。
- **ルーム参加**: ルームIDを直接入力して参加。
- **ルーム検索**: 公開ルーム一覧を検索・表示し、選択して参加。

### 3.2 ゲーム画面 (roomId が存在する場合)
- **サイドバー**:
  - プレイヤー一覧（現在のターン者はハイライト）。
  - プレイヤースコアのリアルタイム表示。
  - チャット欄とメッセージ入力。
- **メインボード**:
  - **山札**: 残り枚数を表示し、クリックで「カードを引く」。
  - **場 (Field)**: 出されたカードを一覧表示。カードをクリックすると「アクション（スコア加算）」。
  - **手札 (Hand)**: 自分の手札を表示。クリックすると「カードを場に出す」。
- **終了画面**: 勝利したプレイヤーを表示し、「再戦」ボタンを提供。

---

## 4. ターンの管理

- `gameState.turnStartTime` を基準とし、30秒からのカウントダウンを行います（クライアント側のタイマー表示）。
- `isMyTurn`: 現在の `currentTurnIndex` に該当するプレイヤーが自分自身（`myConnectionId`）であるかを判定し、操作の有効・無効を制御します。
